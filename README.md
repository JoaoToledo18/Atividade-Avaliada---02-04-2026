# Projeto de Cobrança

## Descrição
Proposta de um sistema para otimizar o processo de cobrança, utilizando UML (principalmente diagrama de atividades).

## Fluxo
- Recebe o título  
- Envia 1ª carta + ligação (simultâneo)  
- Sem retorno → 2ª carta  
- Sem sucesso → cobrança externa  
- Sem acordo → protesto  
- Pagamento → finaliza processo  

## Casos de Uso
- Cadastrar título  
- Enviar cobrança  
- Realizar contato  
- Negociar pagamento  
- Registrar pagamento  

## Requisitos

### Funcionais
- Cadastro de títulos  
- Envio automático de cartas  
- Controle de negociações  

### Não Funcionais
- Sistema web  
- Seguro e simples
